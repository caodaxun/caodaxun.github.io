dyld加载流程

发表于 2020-12-07

dyld

dyld ：动态库链接器，负责加载程序和程序依赖的动态库，内核读取 Mach-O 文件后将读取内容交给 dyld 加载，dyld 加载完毕后才会执行 main 函数

新建项目，在 main 函数中打断点，通过 bt 查看调用栈，看到 App 启动后会执行 libdyld.dylib 的 start 操

作，这边调用栈的信息比较少

继续在 ViewController 中重写 load 方法 ，load 方法打断点，通过 bt 查看调用栈，发现一个 _dyld_start 的调用

dyld 源码中搜索 _dyld_start ，可以在 dyldStartup.s 文件中找到 _dyld_start 的汇编实现，文件中按照 i386、x86_64、arm64、arm 不同架构做了逻辑处理

查看 arm64 架构下汇编源码

找到关键部分代码，看到 bl 跳转指令，看注释信息，这边会调用 dyldbootstrap::start

dyldbootstrap::start

【1】dyldbootstrap::start

源码中搜索 dyldbootstrap 命名空间的 start 方法

dyldbootstrap::start 主要操作：

  1. 调用 rebaseDyld() dyld 重定位

  2. __guard_setup 栈溢出保护

  3. dyld::_main 进入 dyld 的 _main 函数

dyld::_main

【2】dyld::_main

dyld::_main主要操作：

  1. 主程序运行环境初始化，获取当前运行架构
  2. 加载共享缓存
  3. 实例化主程序，赋值给 ImageLoader::LinkContext
  4. 加载插入的动态库
  5. 链接主程序
  6. 链接动态库
  7. 弱符号绑定
  8. 执行初始化方法
  9. 寻找程序入口 LC_MAIN
  • 分析第 3 步和第 8 步
