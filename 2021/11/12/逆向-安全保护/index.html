数据加密

加密算法：

  • BASE64：一种基于64个可打印字符表示二进制数据的方法，主要为了将二进制数据变成可见字符以便传输，可以解密
  • MD5：将数据运算变成另一个固定长度值，用于数据校验，不可逆
  • DES：一种对称加密算法，用于对二进制数据进行加密，明文、密文和秘钥的分组长度都是64
  • AES：密码学中的高级加密标准，本质上是一种对称分组密码体制，用来代替DES
  • RSA：一种非对称加密算法，存在一个公钥和一个私钥，分别用于加密和解密，速度比对称加密慢很多

对称加密：加密解密使用同一秘钥

非对称加密：存在一个公钥和一个私钥

本地存储加密

本地存储方式：NSUserDefaults、文件、数据库

存储数据时，可以使用哈希算法、对称加密算法（DES/AES）进行加密存储，或者系统 keychain 进行存储，实现加密算法时也不能将 key 以明文形式写在代码中

网络传输加密

HTTPS + 本地证书校验方式

传输过程中可以使用 RSA+AES 进行加密传输

  1. 客户端随机生成 AES 的key
  2. 使用 RSA 公钥加密算法加密随机的 key
  3. 使用随机的 key 对数据进行 AES 加密
  4. 将 RSA 加密后的 key 和 AES 加密后的数据发送给服务器
  5. 给发送的数据加上时间戳，以防止重放攻击

反调试

ptrace

ptrace 提供了一个 PT_DENY_ATTACH 参数，告诉系统阻止调试器附加

sysctl

当目标进程被调试的时候，会有一个标志表明自己正在被调试，可以通过 sysctl 查看当前进程的信息，从这个标志位得到当前的调试状态

syscall

为了实现从用户态切换到内核态，系统提供了 syscall 函数

ptrace 也是通过系统调用去实现的，sys/syscall.h 中可以找到 ptrace 对应的编号

svc

若不想暴露自己的ptrace等系统方法，不想被符号断点中断，可以采用 syscall 的汇编代码来调试 ptrace，实现从用户态到内核态的切换

isatty

检测文件描述符是否连接到一个终端设备，如果是返回1，而 LLDB 调试时需要远程连接 iOS 设备，因此可以用它来做反调试

注入对抗

注入的方式通常使用 DYLD_INSERT_LIBRARIES 环境变量将 dylib 注入

自己编写的 dylib 无法加载到目标应用的时候，表明应用做了反注入保护。

iOS10以下系统

在Xcode的编译设置选项 Other Linker Flags 添加 -Wl,-sectcreate,__RESTRICT,__restrict,/dev/null,

dyld 检测到 MachO中存在 __RESTRICT,__restrict 这样的节（Section）时，DYLD_INSERT_LIBRARIES 环境变量会被忽略，导致注入失败。

反反注入：

  1. 把section的名字修改下（如果主程序同时具备完整性检测那就不行了）
  2. 使用Cydia自带的cynject工具注入，可以无惧主程序完整性检测 /usr/bin/cynject<pid><dylib>

完整性检测

检测目标是否被脱壳、是否被静态注入、内容是否被修改、是否被重签名等

常见方式有加载命令检测、代码段检测、签名信息检测

加载命令检测

为了使未越狱设备也能顺利使用插件，通常会在 LoadCommand（加载命令）中增加一个 LC_LOAD_DYLIB 或者 LC_LOAD_WEAK_DYLIB 的方式来指向自己的动态库，从而实现注入功能，这种方式通常也称为静态注入

加载命令检测即检测 Mach-O 文件的 LoadCommand 中 LC_LOAD_DYLIB 或 LC_LOAD_WEAK_DYLIB 是否存在异常

代码段检测

检测 __TEXT,__text 是否被修改（当然也可以检测其它 Section）

&_mh_execute_header 指向 MachO 在内存中的起始位置，getsectiondata返回指定 Section 的内存地址大小。实际应用中可以将正确的 Hash 值存到 服务器或者加密后放到本地，在进程启动时进行对比，不匹配则可以进行一些逻辑处理

签名信息检测

检测 LC_CODE_SIGNATURE 中的特征信息，比如 TeamID是否和用户匹配
